# ✅ Fixed On-Top Coupon Assignment & Validation Logic (2026-02-27)

## ผลการแก้ไข (สรุปล่าสุด)
- ✅ **Proper Coupon Slots**: แก้ไขปัญหาที่คูปองเสริม (On-top) จากหน้ากระเป๋าคูปอง เด้งไปใส่ผิดช่องในหน้าจอง โดยการปรับแต่ง `useBookingLogic` ให้เช็ค Property `category` === 'ONTOP' ก่อนยัดลง State `manualOntopCoupon`
- ✅ **Dynamic Validation logic**:
  - เปลี่ยนวิธีเช็คยอดขั้นต่ำ (Minimum Spend): คูปองหลักจะเช็คจากยอดรวมบิล (`originalPrice`) ส่วนคูปองเสริมจะเช็คจากยอดหลังหักส่วนลดคูปองหลัก (`priceAfterMain`) เพื่อให้สอดคล้องกับ Business Logic
  - แก้ไข UI (แถบด้านล่าง): เปลี่ยนการเช็คความValid ของคูปองจากการตีความ `discount === 0` (ซึ่งทำให้คูปองแถมของฟรีถูกมองเป็น Invalid) ไปเป็นการใช้ตัวแปร `isCouponInvalid` จาก Logic จริงแทน 
- ✅ **Multi-Coupon Rendering**: แถบสรุปการจองถูกอัปเดตให้สามารถแสดงชื่อคูปองทั้งหมดที่ถูกใช้งาน (หลัก + เสริม) ต่อกันด้วยเครื่องหมาย `+` อย่างชัดเจน
